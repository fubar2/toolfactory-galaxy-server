Demonstrations are in the ToolFactory sample history. It should appear when you login.
If it is not visible in the right panel when you login, check the history list and if necessary, install it:
  1. Copy this URL: https://zenodo.org/record/4729971/files/TF_demo_history_April30.tar.gz?download=1
  2. Select the "User" tab of the top bar; Select Histories; Select Import; Paste that URL into the History import upload box
  3. Switch to the imported history for sample ToolFactory jobs to illustrate some of the kinds of tools and features.
Learning to use the ToolFactory: Make changes to the samples and try the resulting new tools
Re-run a tool generation job using the generated XML wrappers using the circular arrow "Rerun" button to begin exploring. The ToolFactory form that generated that tool will appear, filled in the way it was when the sample (installed in the ToolFactory Generated Tools section) was generated.
Change the script, the wording of prompts and add i/o files. The updated tool is immediately available to run so you can see how they change for users. Experimenting with the examples may help you figure out how to use it for your work.
The Hello World example is a useful base model. Any working bash script with a few parameters can be substituted starting with that example as a model. Add parameters and outputs/inputs as needed.
Generated tools are ordinary Galaxy tools. They work for any user and in workflows.
Generating a tool takes a few seconds. Testing takes longer when it is requested. Testing with Planemo runs in a separate container for technical reasons. Planemo is run twice and a Galaxy is built and torn down each time. The first run is to generate test outputs that are added to the new testedtool archive. The second time those sample outputs are tested. Conda adds time to install the dependencies and then there's the actual runtime for the test data
Tested tools and reports are in the exported ../compose/export/galaxy/tested_TF_archives and ../tested_TF_reports directories on the local machine.
Examples in the history and installed in the Appliance.

  • Hello_toolfactory runs a bash script with a single user supplied parameter - "World" for example. STDOUT goes to the history
  • Tacrev uses STDOUT and STDIN without parameters.
  • Two python scripts to reverse lines in a text file use (pyrevargparse) argparse style and with (pyrevpos) positional parameters
  • The perl GC counter takes positional parameters and is taken straight from the Planemo documentation tool examples
  • The bwa example is from the Planemo documentation. There are 2 implementations. One uses the Planemo tutorial templating code to demonstrate a command over-ride. An equivalent tool uses bash and positional parameters. Python or bash scripting will allow some tool builders to script the same logic as a skilled developer using Galaxy tool templating for complicated pre-processing.
  • The Sed example shows how unix utilities like sed - or similar Conda package - can be driven using positional parameters
  • The plotter tool illustrates the use of collections for messy outputs.
  • The repeats demonstration shows how any number of repeating parameters can be selected and used in a tool
  • There are two versions of a planemo lint tool - a simple bash script and a much more complex python script to show how some Conda dependency pre-processing complexities can be handled in different ways
  • The Evil_script_that_fails example tries to create a file where it should not. It should build ok but an error should be noted in the history tgz after it runs. Note that it can do evil under docker because it runs as root - but only evil within the docker container itself. We hope.

Warning: If 2+ tools needing new dependencies are run for the first time at the same time, Conda may break!

The demonstration tools are relatively simple for a good reason. Galaxy's UI is not ideal for the purpose of building tools with dozens of parameters. It can be done but there will be plenty of cases where the old fashioned manual/Planemo way will be way easier for most people. However, if you have the patience, it works. The ToolFactory is ideal for relatively simple scripts with a handful of files and parameters.
